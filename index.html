软件操作指南与核心理念

核心理念

本工具旨在帮助非技术背景的产品经理，通过 AI 辅助将模糊的需求转化为 **Google AI Studio** 可直接执行的完备代码生成指令 (Build Prompt)。

一、双重记忆系统

  • 显性记忆 (项目知识库): 右上角的黄色记事本。记录项目的“宪法”（如 API Key、技术栈）。AI 会自动追加，您也可修改。
  • 隐形记忆 (Content A): 系统会自动在后台记录您的每一次修改指令、每一次问答选择、每一次采纳建议。这是生成最终 Prompt 的关键素材。

二、终极 Prompt 生成机制 (三步法)

点击“生成最终 Prompt”时，AI 将执行海量运算：

  1. 整理历史 (Content A -> Content B): AI 产品总监整理所有隐形历史，生成一份详尽的需求演变史。
  2. 架构设计 (Map + KB -> Content C): AI 架构师基于当前的思维导图和知识库，生成技术规格书。
  3. 终极融合 (B + C -> Content D): AI 总设计师将需求史与技术规格融合，生成最终指令。
  4. 最终产出: Content B (需求全貌) + Content D (构建指令)。
